1. What is CSS?

CSS stands for Cascading Style Sheets. It is a stylesheet language used to describe the presentation and visual appearance of HTML documents. While HTML creates the structure of a webpage, CSS controls how that structure looks including colors, fonts, spacing, layout, and responsiveness across different screen sizes.

CSS works by selecting HTML elements and applying styles to them. The term "cascading" refers to the way CSS determines which styles to apply when multiple rules target the same element. Styles can cascade from multiple sources including browser defaults, external stylesheets, internal stylesheets, and inline styles.

🧠 The Web Development Trio:
🏗️ HTML = Structure (what things are)
 🎨 CSS = Presentation (how things look)
 ⚡ JavaScript = Behavior (how things work)

2. Adding CSS to HTML

There are three ways to add CSS to an HTML document. Each method has its own use case, but external CSS is considered the best practice for most projects because it separates concerns and makes code easier to maintain.

📋 THREE WAYS TO ADD CSS
<!-- 1. INLINE CSS (inside HTML element using style attribute) -->
<p style="color: red; font-size: 16px;">Red text</p>

<!-- 2. INTERNAL CSS (inside <style> tag in the <head>) -->
<head>
    <style>
        p {
            color: blue;
            font-size: 18px;
        }
    </style>
</head>

<!-- 3. EXTERNAL CSS (separate .css file - BEST PRACTICE!) -->
<head>
    <link rel="stylesheet" href="styles.css">
</head>
✅ Best Practice: Always use External CSS for production websites. It keeps HTML clean, allows reuse across multiple pages, and makes maintenance easier.

3. CSS Selectors

CSS selectors are patterns used to select and target HTML elements that you want to style. Understanding selectors is fundamental to writing effective CSS. Selectors can target elements by their tag name, class, ID, attributes, or their relationship to other elements.

🎯 ALL SELECTOR TYPES
/* Universal Selector - selects everything */
* { margin: 0; padding: 0; }

/* Element Selector - selects all elements of that type */
p { color: blue; }
h1 { font-size: 32px; }

/* Class Selector - MOST COMMON (starts with .) */
.highlight { background: yellow; }
.btn-primary { background: blue; color: white; }

/* ID Selector - unique element (starts with #) */
#header { height: 80px; }

/* Multiple Selectors */
h1, h2, h3 { font-family: sans-serif; }

/* Descendant Selector - elements inside another */
article p { line-height: 1.6; }

/* Child Selector - direct children only */
nav > a { text-decoration: none; }

/* Pseudo-classes - element states */
a:hover { color: red; }
input:focus { border-color: blue; }
li:first-child { font-weight: bold; }
li:nth-child(odd) { background: #f0f0f0; }

/* Pseudo-elements - parts of elements */
p::first-line { font-weight: bold; }
h1::before { content: "👉 "; }

4. Colors & Backgrounds

Colors in CSS can be specified using multiple formats including named colors, hexadecimal codes, RGB, RGBA, HSL, and HSLA. Each format has advantages depending on the use case.

🎨 COLORS & BACKGROUNDS
/* Named Colors */
color: red;
color: dodgerblue;
color: tomato;

/* Hex Codes (MOST COMMON) */
color: #FF0000;      /* Red */
color: #6366f1;      /* Indigo */
color: #10b981;      /* Green */
color: #000000;      /* Black */
color: #FFFFFF;      /* White */

/* RGB / RGBA (a = alpha/transparency 0-1) */
color: rgb(255, 0, 0);
color: rgba(255, 0, 0, 0.5);   /* 50% transparent red */

/* HSL (Hue, Saturation, Lightness) */
color: hsl(0, 100%, 50%);       /* Red */
color: hsla(240, 100%, 50%, 0.7);

/* Background Properties */
background-color: #f0f0f0;
background-image: url('image.jpg');
background-size: cover;
background-position: center;
background-repeat: no-repeat;

/* Gradients */
background: linear-gradient(to right, red, blue);
background: linear-gradient(135deg, #6366f1, #8b5cf6);
background: radial-gradient(circle, white, black);

5. The Box Model

The CSS Box Model is the foundation of layout in CSS. Every HTML element is a rectangular box consisting of four parts: content, padding, border, and margin. Understanding how these parts interact is essential for creating proper layouts.

📦 THE BOX MODEL
div {
    /* Content - the actual content area */
    width: 300px;
    height: 200px;
    
    /* Padding - space between content and border */
    padding: 20px;              /* All sides */
    padding: 10px 20px;         /* Top/Bottom, Left/Right */
    padding: 10px 15px 20px 25px; /* Top, Right, Bottom, Left */
    
    /* Border - the edge line */
    border: 2px solid #333;
    border-width: 2px;
    border-style: solid;  /* solid, dashed, dotted, double */
    border-color: #333;
    border-radius: 8px;   /* Rounded corners! */
    
    /* Margin - space OUTSIDE the element */
    margin: 30px;
    margin: 20px auto;    /* Auto = center horizontally */
}

/* Total Width Calculation:
   width + padding-left + padding-right + border-left + border-right + margin-left + margin-right
=  300   + 20 + 20             + 2 + 2                    + 30 + 30
=  404px total */

/* Box-Sizing Fix (USE THIS!) */
* {
    box-sizing: border-box;  /* Width includes padding and border */
}

8. Flexbox Layout

Flexbox is a one-dimensional layout system designed for arranging items in rows or columns. It provides powerful alignment capabilities and makes it easy to distribute space between items.

📐 FLEXBOX CHEAT SHEET
/* Container Properties */
.container {
    display: flex;
    flex-direction: row;           /* row | column | row-reverse */
    justify-content: center;       /* Horizontal alignment */
    /* options: flex-start, center, flex-end, space-between, space-around, space-evenly */
    
    align-items: center;           /* Vertical alignment */
    /* options: flex-start, center, flex-end, stretch, baseline */
    
    flex-wrap: wrap;               /* Allow items to wrap to next line */
    gap: 20px;                     /* Space between items */
}

/* Item Properties */
.item {
    flex: 1;            /* Grow equally - shorthand for flex-grow */
    flex: 2;            /* Take twice the space of other items */
    flex-grow: 0;       /* Don't grow */
    flex-shrink: 0;     /* Don't shrink */
}

/* PERFECT CENTERING */
.centered {
    display: flex;
    justify-content: center;
    align-items: center;
    height: 100vh;      /* Full viewport height */
}

9. CSS Grid

CSS Grid is a two-dimensional layout system for rows AND columns simultaneously. It is the most powerful layout system in CSS and is perfect for complex page layouts.

📊 CSS GRID
.grid-container {
    display: grid;
    grid-template-columns: 1fr 1fr 1fr;  /* 3 equal columns */
    grid-template-columns: repeat(3, 1fr);  /* Same as above */
    grid-template-columns: 200px 1fr 2fr;   /* Mixed sizes */
    
    /* Responsive grid (auto-fit magic!) */
    grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
    
    gap: 20px;                    /* Row and column gap */
    row-gap: 20px;
    column-gap: 20px;
}

/* Item Placement */
.item {
    grid-column: 1 / 3;           /* Start at line 1, end at line 3 (span 2) */
    grid-column: span 2;          /* Same as above */
    grid-row: 1 / 3;              /* Span 2 rows */
}

/* Perfect Centering with Grid */
.center-grid {
    display: grid;
    place-items: center;          /* Center BOTH horizontally & vertically */
    height: 100vh;
}
💡 When to use what:
Use Flexbox for 1D layouts (navbar, centering items, row of cards)
Use Grid for 2D layouts (entire page layout, image gallery, dashboard)

10. Responsive Design

📱 MEDIA QUERIES
/* Mobile-First Approach */
.container { width: 100%; padding: 1rem; }

/* Tablet (768px+) */
@media (min-width: 768px) {
    .container { max-width: 720px; margin: 0 auto; }
}

/* Desktop (1024px+) */
@media (min-width: 1024px) {
    .container { max-width: 960px; }
}

/* Large screens */
@media (min-width: 1280px) {
    .container { max-width: 1200px; }
}

/* Responsive Grid Example */
.card-grid {
    display: grid;
    grid-template-columns: 1fr;              /* 1 column on mobile */
    gap: 1rem;
}

@media (min-width: 640px) {
    .card-grid { grid-template-columns: repeat(2, 1fr); }  /* 2 columns */
}

@media (min-width: 1024px) {
    .card-grid { grid-template-columns: repeat(3, 1fr); }  /* 3 columns */
}

CSS Units for Responsive Design: Use rem for fonts, % for widths, vh/vw for viewport-based sizing, and clamp() for fluid sizing.

11. CSS Animations

✨ ANIMATIONS
/* Transitions - smooth changes */
button {
    background: blue;
    transition: all 0.3s ease;
}
button:hover {
    background: red;
    transform: scale(1.1);
}

/* Keyframe Animations */
@keyframes slideIn {
    from {
        opacity: 0;
        transform: translateY(30px);
    }
    to {
        opacity: 1;
        transform: translateY(0);
    }
}

.element {
    animation: slideIn 0.6s ease forwards;
    animation-delay: 0.2s;
    animation-iteration-count: infinite;
    animation-direction: alternate;
}

/* Transforms */
transform: rotate(45deg);
transform: scale(1.5);
transform: translate(50px, -20px);
transform: skew(10deg);

✅ CSS Best Practices

✅ DO THIS
  • Use external stylesheets
  • Follow mobile-first approach
  • Use classes, not IDs for styling
  • Use Flexbox/Grid for layouts
  • Use rem/em for font sizes
  • Keep specificity low
  • Use CSS variables for repeated values
❌ AVOID THIS
  • Inline styles
  • Excessive !important
  • Deeply nested selectors
  • Fixed pixel sizes everywhere
  • IDs for styling
  • Tables for layout
  • Deprecated properties
