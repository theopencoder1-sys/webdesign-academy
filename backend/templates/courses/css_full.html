Separate Style from Content

The driving problem behind CSS is simple: early HTML allowed you to override browser defaults with attributes like <table border="2" bordercolor="black">, but this became unmanageable. CSS solves this by separating what to display (HTML content) from how to display it (formatting information in .css files).

This gives you the DRY Principle (Don't Repeat Yourself): define style information once and reuse it everywhere. Want to change the company's brand colour? Update one line in your CSS file, not every single HTML tag.

Anatomy of a Rule

How CSS Rules Work

Every style sheet contains one or more CSS Rules. Each rule has:

body { font-family: Tahoma, Arial, sans-serif; /* Declaration */ color: black; /* Property: Value */ background: white; margin: 8px; } /* ^^^^^^ Selector ^^^^^^^^^^^^^^^^^^^^^^^ Declaration Block */
Selector Reference

Every Way to Target an Element

/* By Tag Name */ h1 { color: red; } /* Targets <h1> */ /* By Class Attribute */ .large { font-size: 16pt; } /* Targets class="large" */ p.large { font-weight: bold; } /* Targets <p class="large"> only */ /* By Element ID */ #p20 { font-weight: bold; } /* Targets id="p20" */
Interactive Selectors

Pseudo Selectors

/* Hover - when mouse is over the element */ p:hover, a:hover { background-color: yellow; } /* Link States */ a:link { color: blue; } /* Unvisited link */ a:visited { color: green; } /* Visited link */
Box Model Deep Dive

The CSS Box Model

Every element is a rectangular box. Understanding this is the most important concept in CSS layout.

Total element width = width + left padding + right padding + left border + right border + left margin + right margin /* Margin and Padding are TRANSPARENT */
Measurement Reference

CSS Distance Units

Absolute Relative
2px — pixels 2em — 2× element's font size
1mm — millimetres 3rem — 3× root element's font size
2cm — centimetres
3pt — printer points (1/72 inch)
Positioning Guide

The position Property

position: static; /* Default — in document flow */ position: relative; /* Offset from normal position via top/right/bottom/left */ position: fixed; /* Fixed to screen (doesn't scroll) */ position: absolute; /* Relative to nearest positioned ancestor */

Fixed position (0,0) is the top-left corner of the screen.

Visibility Control

Show and Hide Elements

display: none; /* Element REMOVED from layout entirely */ display: inline; /* Acts like */ display: block; /* Acts like
*/ display: flex; /* Flex container */ display: grid; /* Grid container */ visibility: hidden; /* Hidden but SPACE STILL HELD */ visibility: visible;/* Normal */
💡 KEY NOTE: Use display: none when you want the element GONE. Use visibility: hidden when you want to hide it but keep the layout space.
Advanced Concept

Inheritance & Specificity

Some properties (like font-size) are inherited from parent elements. Others (like border) are not.

When multiple rules match an element, the most specific rule wins:

span { color: red; } /* Less specific */ span.test { color: green; } /* More specific — WINS */
Pro Tips

CSS in the Real World

Professional developers use CSS preprocessors (like Less, Sass) to add variables and functions, making large stylesheets easier to maintain. Composition can be tricky — it's sometimes hard to figure out which rule from which stylesheet is causing issues. Use browser DevTools (F12) to inspect and debug!

1. What is CSS?

CSS (Cascading Style Sheets) controls how HTML elements look. If HTML is the skeleton, CSS is the skin, clothes, and makeup.

✅ What CSS Can Do

  • Colors & Backgrounds
  • Fonts & Typography
  • Spacing (Margin/Padding)
  • Borders & Shadows
  • Layouts (Flexbox/Grid)
  • Animations & Transitions
  • Responsive Design
  • Pseudo-classes (:hover)

❌ Common Mistakes

  • Inline styles everywhere
  • Overusing !important
  • Deeply nested selectors
  • Fixed pixel sizes for everything
  • Not using semantic classes
  • Ignoring mobile responsiveness

2. Adding CSS to HTML

<!-- 1. INLINE (AVOID!) -->
<p style="color: red; font-size: 16px;">Red text</p>

<!-- 2. INTERNAL (OK for small pages) -->
<head><style> p { color: blue; } </style></head>

<!-- 3. EXTERNAL (BEST PRACTICE!) -->
<head><link rel="stylesheet" href="styles.css"></head>

3. CSS Selectors - Complete Reference

/* Universal */ * { margin: 0; padding: 0; }
/* Element */ p { color: blue; }  h1 { font-size: 32px; }
/* Class (MOST USED) */ .highlight { background: yellow; }
/* ID */ #header { height: 80px; }
/* Multiple */ h1, h2, h3 { font-family: sans-serif; }
/* Descendant */ article p { line-height: 1.6; }
/* Child */ nav > a { text-decoration: none; }
/* Adjacent Sibling */ h1 + p { margin-top: 0; }
/* Attribute */ input[type="text"] { border: 1px solid gray; }
/* Pseudo-class */ a:hover { color: red; }
/* Pseudo-class */ li:first-child { font-weight: bold; }
/* Pseudo-class */ li:nth-child(odd) { background: #f0f0f0; }
/* Pseudo-class */ li:nth-child(3n) { color: blue; }
/* Pseudo-element */ p::first-line { font-weight: bold; }
/* Pseudo-element */ h1::before { content: "👉 "; }
/* Pseudo-element */ h1::after { content: " ✨"; }

4. Colors & Backgrounds

/* Named */ color: red; color: dodgerblue;
/* Hex (MOST COMMON) */ color: #FF0000; color: #6366f1; color: #10b981;
/* RGB */ color: rgb(255, 0, 0);
/* RGBA (transparency) */ color: rgba(255, 0, 0, 0.5);
/* HSL */ color: hsl(0, 100%, 50%);

/* Backgrounds */
background-color: #f0f0f0;
background-image: url('bg.jpg');
background-size: cover; background-position: center;
background: linear-gradient(135deg, #6366f1, #8b5cf6);
background: radial-gradient(circle, white, black);

5. The Box Model

/* Content → Padding → Border → Margin (inside → outside) */
div {
    width: 300px;           /* Content area */
    padding: 20px;          /* Space inside border */
    border: 2px solid #333; /* The edge */
    margin: 30px;           /* Space outside */
    border-radius: 8px;     /* Rounded corners! */
}
/* Total width = 300 + 40 + 4 + 60 = 404px */

/* FIX: Include padding & border in width */
* { box-sizing: border-box; } /* USE THIS ALWAYS! */

6. Flexbox - Complete Guide

/* CONTAINER */
.container {
    display: flex;
    flex-direction: row;           /* row | column | row-reverse */
    justify-content: center;       /* start | center | end | space-between | space-around | space-evenly */
    align-items: center;           /* start | center | end | stretch | baseline */
    flex-wrap: wrap;               /* nowrap | wrap | wrap-reverse */
    gap: 20px;                     /* Space between items */
}
/* ITEMS */
.item { flex: 1; }      /* Grow equally */
.item { flex: 2; }      /* Take 2x space */
.item { flex-shrink: 0; } /* Don't shrink */

/* PERFECT CENTERING */
.centered { display: flex; justify-content: center; align-items: center; height: 100vh; }

7. CSS Grid - Complete Guide

.grid { display: grid;
    grid-template-columns: 1fr 1fr 1fr;          /* 3 equal columns */
    grid-template-columns: repeat(3, 1fr);        /* Same */
    grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)); /* Responsive! */
    gap: 20px; }

.item { grid-column: 1 / 3; }       /* Span 2 columns */
.item { grid-column: span 2; }      /* Same */
.item { grid-row: 1 / 3; }          /* Span 2 rows */

/* Perfect Grid Centering */
.center { display: grid; place-items: center; height: 100vh; }

8. Responsive Design

/* Mobile First! */
.container { width: 100%; padding: 1rem; }

/* Tablet */ @media (min-width: 768px) { .container { max-width: 720px; margin: 0 auto; } }
/* Desktop */ @media (min-width: 1024px) { .container { max-width: 960px; } }
/* Large */ @media (min-width: 1280px) { .container { max-width: 1200px; } }

/* Responsive Grid */
.cards { display: grid; grid-template-columns: 1fr; gap: 1rem; }
@media (min-width: 640px) { .cards { grid-template-columns: repeat(2, 1fr); } }
@media (min-width: 1024px) { .cards { grid-template-columns: repeat(3, 1fr); } }

/* Responsive Fonts */
h1 { font-size: clamp(1.5rem, 5vw, 3rem); } /* Min, Preferred, Max */

9. CSS Animations

/* Transitions */
.btn { background: blue; transition: all 0.3s ease; }
.btn:hover { background: red; transform: scale(1.1); }

/* Keyframe Animations */
@keyframes slideIn {
    from { opacity: 0; transform: translateY(30px); }
    to { opacity: 1; transform: translateY(0); }
}
.element { animation: slideIn 0.6s ease forwards; }

/* Transforms */
transform: rotate(45deg); transform: scale(1.5);
transform: translate(50px, -20px); transform: skew(10deg);

✅ CSS Best Practices Checklist

  • ✅ Use external stylesheets
  • ✅ Mobile-first approach
  • ✅ Use classes, not IDs
  • ✅ Flexbox/Grid for layout
  • ✅ rem/em for fonts
  • ✅ CSS variables for colors
  • ❌ Inline styles
  • ❌ Excessive !important
  • ❌ Deeply nested selectors
  • ❌ Fixed px everywhere
  • ❌ Tables for layout
  • ❌ Ignoring accessibility
