Chapter 1: What is HTML?

HTML (HyperText Markup Language) is the foundation of the World Wide Web. Created by Tim Berners-Lee in 1991, HTML is not a programming language — it is a markup language that tells browsers how to structure and display content.

Key Concepts:

  • HyperText — Text that contains links to other texts/webpages
  • Markup — Using tags to "mark up" content with structure
  • Language — A standardized syntax understood by web browsers

Every website you visit — from Google to YouTube to this platform — uses HTML as its backbone. HTML works with CSS (for styling) and JavaScript (for interactivity) to create the complete web experience.

📌 Note: The current version is HTML5, released in 2014. It added semantic elements, audio/video support, and better form controls.

Chapter 2: Basic Document Structure

Every HTML document follows a standard structure. Here is the minimum required code:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Web Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
    <p>This is my first web page.</p>
</body>
</html>

Breaking Down Each Part:

TagPurposeRequired?
<!DOCTYPE html>Tells browser this is HTML5Yes
<html>Root element - wraps everythingYes
<head>Metadata, title, styles, scriptsYes
<meta charset>Character encoding (UTF-8)Recommended
<meta viewport>Responsive design on mobileRecommended
<title>Browser tab textYes
<body>All visible contentYes

Chapter 3: HTML Elements & Tags

An HTML element consists of an opening tag, content, and a closing tag. Some elements are void elements (self-closing).

Paired Tags (Most Common):

<p>This is a paragraph element.</p>
<h1>This is a heading element.</h1>
<div>This is a division element.</div>
<a href="page.html">This is a link element.</a>

Void (Self-Closing) Elements:

<br>        <!-- Line break -->
<hr>        <!-- Horizontal rule -->
<img src="photo.jpg" alt="Description">  <!-- Image -->
<input type="text" placeholder="Name">   <!-- Form input -->
<meta charset="UTF-8">  <!-- Metadata -->

Nesting Elements:

Elements can (and should) be nested inside each other. Proper nesting means closing tags in the reverse order they were opened.

<!-- CORRECT Nesting -->
<div>
    <h2>Title</h2>
    <p>Some <strong>bold</strong> text here.</p>
</div>

<!-- WRONG Nesting -->
<div><h2>Title</div></h2>  <!-- Never do this! -->

Chapter 6: Headings & Paragraphs

Headings create document hierarchy. There are 6 levels: <h1> (most important) to <h6> (least important).

<h1>Main Page Title - Use ONLY ONCE per page!</h1>
<h2>Major Section</h2>
<h3>Sub-section</h3>
<h4>Sub-sub-section</h4>
<h5>Minor Heading</h5>
<h6>Smallest Heading</h6>

<p>Paragraphs are block-level elements. Browsers automatically add margin before and after paragraphs.</p>
<p>Multiple spaces and line breaks in your code are collapsed into a single space when displayed.</p>

<br>  <!-- Line break - starts a new line -->
<hr>  <!-- Horizontal rule - creates a divider line -->
⚠️ SEO Rule: Use exactly ONE <h1> per page. Don't skip heading levels (h1 → h2 → h3, never h1 → h3).

Chapter 8: Links & Navigation

The <a> (anchor) tag creates hyperlinks. The href attribute specifies the destination.

<!-- External Link -->
<a href="https://google.com">Visit Google</a>

<!-- Open in New Tab (use rel for security!) -->
<a href="https://twitter.com" target="_blank" rel="noopener noreferrer">Twitter</a>

<!-- Internal Link (same site) -->
<a href="/about.html">About Us</a>
<a href="/contact.html">Contact</a>

<!-- Jump to Section (same page) -->
<a href="#chapter-8">Jump to Chapter 8</a>

<!-- Email Link -->
<a href="mailto:hello@example.com">Send Email</a>

<!-- Phone Link (mobile!) -->
<a href="tel:+254700123456">Call Us</a>

<!-- WhatsApp (Kenya essential!) -->
<a href="https://wa.me/254700123456" target="_blank">Chat on WhatsApp 📱</a>

<!-- Download Link -->
<a href="/files/resume.pdf" download>Download Resume</a>

Chapter 9: Images Deep Dive

<!-- Basic Image -->
<img src="nairobi.jpg" alt="Nairobi skyline at sunset">

<!-- With Dimensions -->
<img src="logo.png" alt="Company Logo" width="200" height="100">

<!-- Responsive Image (BEST!) -->
<img src="hero.jpg" alt="Banner" style="max-width:100%; height:auto;">

<!-- Image Formats -->
JPEG  - Photos (small file size)
PNG   - Logos, transparency needed
SVG   - Icons, scalable (never gets blurry!)
WebP  - Modern format (smaller than JPEG/PNG)

<!-- Image as Link -->
<a href="/about">
    <img src="profile.jpg" alt="Click for about page">
</a>

<!-- Lazy Loading (performance!) -->
<img src="photo.jpg" alt="..." loading="lazy">
⚠️ ALWAYS include alt text! Screen readers use it. Google uses it for SEO. It shows if image fails to load.

Chapter 11: Forms & Input Elements

Forms collect user data. The <form> tag wraps all form elements. action specifies where data goes, method is GET or POST.

<form action="/submit" method="POST">
    
    <!-- Text Input -->
    <label for="name">Full Name:</label>
    <input type="text" id="name" name="name" placeholder="Brian Ochieng" required>
    
    <!-- Email -->
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    
    <!-- Password -->
    <label for="pass">Password:</label>
    <input type="password" id="pass" name="password" minlength="8">
    
    <!-- Phone (for M-Pesa!) -->
    <label for="phone">M-Pesa Number:</label>
    <input type="tel" id="phone" name="phone" placeholder="0712345678">
    
    <!-- Dropdown -->
    <label for="course">Course:</label>
    <select id="course" name="course">
        <option value="">-- Select --</option>
        <option value="html">HTML5</option>
        <option value="css">CSS3</option>
        <option value="js">JavaScript</option>
    </select>
    
    <!-- Radio Buttons -->
    <fieldset>
        <legend>Payment Method:</legend>
        <input type="radio" id="mpesa" name="payment" value="mpesa" checked>
        <label for="mpesa">M-Pesa</label>
        <input type="radio" id="card" name="payment" value="card">
        <label for="card">Bank Card</label>
    </fieldset>
    
    <!-- Checkboxes -->
    <input type="checkbox" id="terms" name="terms" required>
    <label for="terms">I agree to terms</label>
    
    <!-- Textarea -->
    <label for="msg">Message:</label>
    <textarea id="msg" name="message" rows="4" placeholder="Your message..."></textarea>
    
    <!-- Submit Button -->
    <button type="submit">Submit</button>
    <input type="submit" value="Submit">
    
</form>

All Input Types:

text, password, email, number, tel, url, date, time, datetime-local, color, range, file, hidden, checkbox, radio, submit, reset, button

Chapter 12: Semantic HTML5

Semantic elements describe their meaning to browsers and developers. Better for SEO, accessibility, and code readability.

✅ Semantic Tags:
  • <header> - Top section
  • <nav> - Navigation
  • <main> - Main content
  • <article> - Independent content
  • <section> - Thematic grouping
  • <aside> - Sidebar
  • <footer> - Bottom section
  • <figure> - Image with caption
  • <time> - Date/time
  • <address> - Contact info
❌ Non-Semantic:
  • <div> - Generic container
  • <span> - Generic inline

These are fine when no semantic tag fits, but use semantic tags first!

✅ Best Practices:
  • Use semantic HTML5 tags
  • Always include alt on images
  • Proper heading hierarchy
  • Include viewport meta tag
  • Close all tags
  • Use lowercase tag names
  • Indent nested elements
  • Validate your HTML
  • Add lang attribute to html
  • Use meaningful title tags
❌ Common Mistakes:
  • Skipping heading levels
  • Using divs for everything
  • Forgetting to close tags
  • Inline styles everywhere
  • Missing alt text
  • No mobile viewport
  • Tables for layout
  • Deprecated tags
  • Missing form labels
  • No document language
